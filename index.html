1 — Information Architecture

Why this grouping? Admins think in terms of what they're protecting (access, identity, traffic) — not the underlying technology. The four groups map to natural security workflows. Moving items from the old "Tools" menu into Settings creates one single destination for all configuration, eliminating the cognitive split between two menus.

2 — Sidebar Tab Order & Rationale

1. Login as its own top-level menu — login security (Mask Login, 2FA, Password Rules, Session Protection, CAPTCHA) is the #1 admin concern. Making it a dedicated menu item gives it the prominence it deserves, separate from network-level Firewall rules. Admins think: "How do users get in?" → Login.

2. Security Policies (Settings) — HTTP security headers are a distinct, set-once concern. Isolated under Settings for clean separation from active firewall rules.

3. Alerts & Reports split — Alerts are real-time event-driven notifications (lockouts, threats, file changes). Reports are scheduled periodic summaries (weekly scans, firewall stats). Each has its own recipients because the person who handles emergencies (CTO/DevOps) is often different from whoever reviews weekly reports (Project Manager/Client).

4. General/Configs/Data/Accessibility — plugin-level meta-settings at the bottom, matching the WPMU DEV ecosystem pattern (Smush, Hummingbird, etc.).

4 — Design Decisions Summary

DecisionRationale
Eliminate "Tools" menu entirelyOne destination for all config — no guessing. Follows Hick's Law: fewer choices → faster decisions.
Section labels in sidebar12 tabs without grouping = choice overload. Labels act as visual anchors — scan group first, then drill into tabs.
Login as own menu itemMost frequently configured area (80%+ support tickets relate to login). Dedicated menu = instant discoverability, no nesting.
Security Headers isolatedHTTP-level protection is a different security layer than app-level auth. Mixing creates category confusion.
General group at bottomMatches Smush, Hummingbird, and all WPMU DEV plugins. Consistency across ecosystem = zero relearning.
"NEW" badges on 3 tabsGuides existing users to relocated features without disruption. Auto-expire after 30 days or first visit.
Two-column content layoutMatches Smush's side-by-side pattern. Reduces scroll depth while keeping related settings visible together.
